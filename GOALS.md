## Долгосрочные цели

### 1. Масштаб модели: 70B+ параметров

Сейчас: крошечная модель (~100M параметров)
Цель: **70 миллиардов+ параметров** в одной RTX 3090

Это возможно благодаря:
- Экстремальному сжатию TRILIX (0.0048 BPW)
- Каждый параметр кодируется ~1 битом вместо 16 бит (FP16)
- 256 кодбуков экспертов для специализации
- Прогрессивное обучение с ростом модели

**Дорожная карта масштабирования:**
```
Фаза 1 (сейчас):  ~100M  параметров  →  отладка архитектуры
Фаза 2 (1 мес):  ~1B    параметров  →  проверка масштабируемости
Фаза 3 (3 мес):  ~13B   параметров  →  конкурент с LLaMA 3.1 8B
Фаза 4 (6 мес):  ~70B+  параметров  →  побить лидеров
```

### 2. Контекстное окно: 1,000,000+ токенов

Сейчас: `max_position_embeddings = 4096` (всего 4K)
Цель: **1,000,000 токенов** (1M), стремиться к 10M+

Реализация:
- Линейный attention → LongRoPE / YaRN / Positional Interpolation
- Sparse attention для длинных контекстов
- Hierarchical KV-cache (этажи памяти: L1=рабочая, L2=недавняя, L3=архивная)
- Прогрессивное удлинение окна: 4K → 32K → 256K → 1M → 10M

**Дорожная карта контекста:**
```
4K    → 32K    → 256K  → 1M    → 10M
[сейчас] [1 мес] [3 мес] [6 мес] [1 год]
```

### 3. VRAM: Минимум занятой памяти

Цель: 70B модель в 24GB VRAM (RTX 3090)

Техники:
- Три уровня сжатия (TRILIX): ~98.5% BPW
- fp8 / int4 кванование для масок внимания
- Gradient checkpointing (не хранить все активации)
- Offloading: тяжёлые компоненты в CPU RAM если нужно
- Эластичный батчсайз: подстраиваться под свободную VRAM

### 1. Soul Codebook: от 1024 к 10,000+ агентам с навыками

**Сейчас:**
- 1024 агента, каждый = 1 обучаемый вектор
- `SOUL.md` — номинальный файл (только описание)
- Навыков НЕТ

**Цель:** настоящая система агентов с навыками

Архитектура агента:
```
Агент = Soul Vector + Skills Stack + Meta-Skill Creator

Soul Vector:    [rank=100] обучаемый вектор "характера"
Skills Stack:   список скиллов (read, write, debug, translate...)
Meta-Skill:     SKILL.md_CREATOR — создаёт новые скиллы на лету
```

**SOUL.md становится ACTUAL файлом:**
```yaml
# soul/agent_007.yaml
id: 7
name: Python Architect
specialization: python_design
soul_vector: [0.12, -0.34, ..., 0.28]  # 100 чисел, обучается
skills:
  - python_expert:    level=5  # встроенные скиллы
  - system_design:    level=4
  - code_review:      level=4
  - architecture:     level=3
created_skills:       []       # скиллы, созданные Meta-Skill Creator
personality:
  creativity: 0.7
  thoroughness: 0.9
  speed: 0.6
```

**Meta-Skill Creator (аналог skill-creator из Anthropic):**
```
Agent получает задачу → анализирует какие скиллы нужны
→ Meta-Skill Creator генерирует новый SKILL.md
→ скилл добавляется в стек агента
→ агент выполняет задачу с новым скиллом
```

**Skill формат (аналог Anthropic skills):**
```yaml
# skills/python_expert/SKILL.md
name: python_expert
trigger: "код на python" | "напиши функцию" | ".py"
actions:
  - write_code
  - debug_code
  - optimize_code
  - test_code
tools:
  - python_interpreter
  - linter
  - unittest
description: >
  Эксперт по Python. Пишет чистый, идиоматичный код.
```

### 2. MoE: 4 эксперта → 256 экспертов

**Сейчас:** 4 эксперта, top-2
**Цель:** 256 экспертов, top-16

Вопрос: **Выиграем ли мы?**

✅ Да, если:
- Каждый эксперт специализируется (math, code, language, reasoning...)
- 256 экспертов создают 256 специализированных кодбуков
- Router становится умнее с MoE-Codebook routing
- Эффективная память на эксперта: ~2KB (кодбук, не копия весов)

⚠️ Риски:
- Router overhead: больше экспертов = сложнее routing
- MoE aux loss нужен для load balancing
- Нужна иерархическая маршрутизация (группы экспертов)

**Реализация:**
```
256 экспертов = 16 групп по 16 экспертов

router_level_1: выбрать группу (16-way)
router_level_2: выбрать эксперта внутри группы (16-way)
total routing: 16 + 16 = 32-way (вместо 256-way)
```

### 3. B3: Differentiable Atom Evolution (DAE)
